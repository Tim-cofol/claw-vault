# 2026-03-03 飞书日历-创建日程（open_id参与人方案）

## 适用场景
- 需要用 Bot/应用能力自动创建飞书日历日程。
- 暂未完成 OAuth（user_access_token + refresh_token）长期授权，无法保证“用户本人是组织者”。
- 目标是：**用户能在自己的日历里看到日程并可加入会议**。

## 核心思路
1) 用 **tenant_access_token（应用身份）** 在机器人主日历上创建 event。
2) 再调用“添加参与人”接口，把目标用户（用 **open_id**）加进 attendees。

> 注意：这种方式下 organizer 可能是机器人/应用日历。若必须“用户是 organizer”，需走 OAuth 拿 user_access_token + refresh_token。

## 必要信息
- 用户 open_id：`ou_e3f2d9fa18f09057b4ffc83c74165f9b`
- 时区：`Asia/Shanghai`

## API 流程（示例）

### 0. 获取 tenant_access_token
`POST https://open.feishu.cn/open-apis/auth/v3/tenant_access_token/internal`

body:
```json
{"app_id":"<APP_ID>","app_secret":"<APP_SECRET>"}
```

### 1. 获取机器人主日历 calendar_id
`POST https://open.feishu.cn/open-apis/calendar/v4/calendars/primary`

Header:
- `Authorization: Bearer <tenant_access_token>`

响应中取：
- `data.calendars[0].calendar.calendar_id`

### 2. 创建日程 event
`POST https://open.feishu.cn/open-apis/calendar/v4/calendars/{calendar_id}/events`

body 示例：
```json
{
  "summary": "讨论质量的设计",
  "description": "议程：\n- 质量目标/指标（SLO）\n- 验收标准\n- 监控与回归策略\n- 责任人",
  "start_time": {"timestamp": "1772672400", "timezone": "Asia/Shanghai"},
  "end_time": {"timestamp": "1772677800", "timezone": "Asia/Shanghai"},
  "need_notification": true
}
```

响应中取：
- `data.event.event_id`
- `data.event.app_link`（有时会因为权限/身份原因打不开，仅做辅助）
- `data.event.vchat.meeting_url`（如有）

### 3. 添加参与人（关键）
`POST https://open.feishu.cn/open-apis/calendar/v4/calendars/{calendar_id}/events/{event_id}/attendees?user_id_type=open_id`

body 示例：
```json
{
  "attendees": [
    {"type": "user", "user_id": "ou_e3f2d9fa18f09057b4ffc83c74165f9b"}
  ]
}
```

## 验收方式
- 用户飞书：日历 → 目标日期 → 对应时间段，能看到该日程。
- 如 app_link 打不开：优先以“日历内可见”为准。
